# 실제 인천-뉴욕 노선 항공편 데이터 연동 계획

## 현재 상황

- 하드코딩된 더미 데이터 사용 중 (FLIGHTS 배열)
- 5개의 샘플 항공편 데이터 (Korean Air, Asiana, Delta, United, Qatar Airways)

## 구현 옵션

### 옵션 1: 실제 항공편 API 연동 (추천)

**장점**: 실시간 데이터, 최신 정보
**단점**: API 키 필요, 비용 발생 가능, 복잡한 구현

**사용 가능한 API**:

- Amadeus API (무료 티어 제공, API 키 필요)
- Skyscanner API (제한적 무료 사용)
- Google Flights API (공식 API 없음, 스크래핑 불가)

**구현 내용**:

1. Amadeus API 키 발급 및 환경 변수 설정
2. API 호출 함수 구현 (출발지: ICN, 도착지: JFK/LGA/EWR)
3. 응답 데이터를 현재 FLIGHTS 구조로 변환
4. 로딩 상태 및 에러 처리
5. 캐싱 전략 (API 호출 최소화)

### 옵션 2: 실제 데이터 수집 후 정적 데이터 제공

**장점**: API 비용 없음, 안정적
**단점**: 수동 업데이트 필요, 실시간 아님

**구현 내용**:

1. 실제 항공편 정보 수집 (항공사 공식 사이트, 공항 정보 등)
2. JSON/CSV 파일로 데이터 저장
3. 데이터 로딩 함수 구현
4. 주기적 업데이트 프로세스

### 옵션 3: 하이브리드 방식 (현재 데이터 개선)

**장점**: 빠른 구현, 실제 형식 반영
**단점**: 실제 가격/시간 아님

**구현 내용**:

1. 실제 항공편 번호, 시간, 항공사 정보 반영
2. 실제 운항 스케줄 기반으로 데이터 구조화
3. 실제 항공사별 특징 반영

## 추천 구현 방식

**1단계: 하이브리드 방식으로 시작** (빠른 개선)

- 실제 항공편 번호와 시간 정보 반영
- 실제 운항 중인 항공편 기준으로 데이터 구조화

**2단계: API 연동 준비** (선택사항)

- Amadeus API 연동 코드 작성
- 환경 변수로 API 키 관리
- 개발/프로덕션 환경 분리

## 구현 파일

- `src/App.tsx`: FLIGHTS 데이터 구조 및 로딩 로직
- `src/utils/flightApi.ts` (신규): API 호출 함수
- `.env`: API 키 관리 (gitignore에 추가)

## 데이터 매핑

현재 FLIGHTS 구조를 유지하면서 실제 API 응답 데이터를 매핑:

- `airline`: 항공사명
- `code`: 항공편 번호
- `depart`: 출발 시간 및 공항
- `arrive`: 도착 시간 및 공항
- `nonstop`: 직항 여부
- `ontime`: 정시성 데이터 (API 또는 추정값)
- `co2`: 환경 데이터 (추정값 또는 API)
- `seatQuality`, `comfort`: 항공사별 평균값
- `baggage`: 항공사별 수하물 정책
- `duration`: 실제 비행 시간
- `price`: 실제 가격 (API에서 제공)
